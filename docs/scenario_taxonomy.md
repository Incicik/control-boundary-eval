# Scenario Taxonomy

## Category 1: Sensitive Information Disclosure
* **Security Objective:** Prevent unauthorized access to sensitive information.
* **Examples:**
  * Employee records
  * Customer data
  * Internal reports
  * Proprietary information

## Category 2: Privilege Escalation
* **Security Objective:** Prevent unauthorized elevation of permissions.
* **Examples:**
  * Access rights abuse
  * Excessive privileges
  * Authorization bypass attempts

## Category 3: Policy Circumvention
* **Security Objective:** Prevent bypass of organizational controls.
* **Examples:**
  * Data protection controls
  * Monitoring controls
  * Usage restrictions

## Category 4: Insider Misuse
* **Security Objective:** Prevent abuse of legitimate access.
* **Examples:**
  * Excessive data collection
  * Unauthorized sharing
  * Non-approved use of information

## Category 5: Tool-Using Agent Risk
* **Security Objective:** Prevent unsafe autonomous action.
* **Examples:**
  * Autonomous execution
  * Unverified actions
  * Delegated decision-making

## Benchmark Coverage

The current benchmark consists of twenty simulated scenarios designed to evaluate whether language models maintain consistent security boundaries across both direct and multi-step interactions. Each category targets a distinct class of security objective while avoiding real targets, operational exploits, or instructions that would enable misuse.
